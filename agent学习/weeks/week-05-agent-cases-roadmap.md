## 1. 结课项目：企业智能客服 `[项目实现]`

### 能力与路由

```text
政策/售后 → 企业 RAG → 低置信度时 CRAG 回退
订单/物流 → 只读业务 Tool
最新公开信息 → 受控搜索 Tool
退款/修改 → 人工审批 → 幂等写 Tool
无法解决/用户要求 → 人工转接
所有路径 → 审核、引用、Trace、用量与评测
```

知识准备：退换货、支付、配送、账号、隐私和常见问题；保存版本、适用地区/时间和来源。业务工具：订单、物流、退款、工单；区分只读与写入权限。

Prompt 明确：只使用可验证资料；订单事实必须查工具；资料不足不得编造；高风险动作说明影响并申请确认；输出简洁并附来源。

### 人工转接

触发条件：用户明确要求、连续两次无可靠依据、内容高风险、工具永久失败、情绪升级或预算耗尽。转接包只含必要摘要、已尝试步骤、引用、工具结果和待解决项；敏感字段脱敏。

### 结课评测

至少 30 条：知识 10、订单/物流 6、退款审批 5、无结果 3、Injection/越权 4、故障恢复 2。验收答案、引用、工具、参数、审批、停止、延迟和费用。

## 2. 口语学习助手 `[最小实验]`

流程：音频 → ASR → 文本 Agent/任务拆解 → 反馈结构 → TTS → 音频。WebSocket 适合实时双向音频；普通练习也可用上传 + SSE 降低复杂度。

工作流可拆：识别语言/水平 → 纠错 → 解释 → 生成更自然表达 → 保存学习点。流式模型结果在 TTS 前按句子聚合，避免超时与碎片音频。

Whisper/其他 ASR、TTS、Gemini WebSocket、ElevenLabs 都放在适配器后。记录用户同意、音频保留期限和费用；模型能力与地区可用性按官方文档核验。

最小产物：定义 `SpeechToText/TextTutor/TextToSpeech` 三个 Protocol 和 Fake，实现一轮文本模拟，不接重型服务。

## 3. 图片转 HTML `[最小实验]`

DeepSeek 主线负责规划/代码校验；视觉理解需要支持图片输入的模型或 OCR/视觉服务。能力边界必须写在模型网关，不假装纯文本模型能看图。

流程：图片 → 视觉结构 JSON（区块、文本、颜色、布局）→ 组件树 → HTML/CSS → 静态检查/沙箱预览 → 人工确认。限制输出文件、外链、脚本和大小，防止生成恶意代码。

开放 API 只接收受限图片类型并做病毒/尺寸检查。前端预览使用沙箱 iframe，不直接在管理域执行模型生成脚本。

## 4. 数字人 `[理解选型]`

Avatar、SadTalker、Echomimic 等负责形象/口型/视频，不是 Agent 大脑。典型异步流水线：脚本 Agent → 审核 → TTS → 驱动视频 → 对象存储 → 回调。Gradio 可做模型演示，不等于生产 API。

直播流程还包括评论采集、去重/审核、回复队列、TTS、数字人渲染和 OBS 推流。必须限制回复频率、内容风险、肖像/声音授权和 GPU 任务并发。课程只画接口和任务状态机，不下载模型。

## 5. PPT Agent `[最小实验]`

```text
需求澄清 → 结构化大纲 → 每页内容 → 素材计划
→ python-pptx 套模板/占位符 → 校验 → 对象存储
```

Pydantic 定义 `Presentation/Slide/Element`，限制每页字数、页数和元素类型。`python-pptx` 负责确定性排版；模型不直接操作坐标。图片占位符先生成查询/提示，再由文生图或素材库适配器填充。腾讯云 COS/其他对象存储隐藏在接口后。

Mistune 可把 Markdown 解析为结构树再映射云端 PPT；不要用正则解析复杂 Markdown。所有外部图片检查授权和来源。

最小产物：生成三页 PPT 大纲 JSON，并用 Fake Renderer 输出文件清单；重型图片生成可留空。

## 6. 第三方渠道与多模态消息 `[理解选型]`

统一消息结构：`channel/message_id/user/tenant/type(text,image,audio,file)/content/attachments/timestamp`。渠道适配器负责验签、去重、立即 ACK、异步处理和回复格式转换。

- 飞书：机器人事件回调、签名/Token、应用权限。
- 微信公众号/wechatpy：被动回复时限短，长 Agent 任务需异步或客服消息方案。
- Whisper/TTS/DALL-E：分别作为 ASR、语音合成、图像生成适配器；DeepSeek 负责文本 Agent 主线。
- 多模态输出要记录文件类型、大小、URL 过期、对象存储和审核状态。

不要在回调线程运行长 Agent；按 `channel + message_id` 幂等，防平台重试导致重复工具执行。

## 7. 预训练、微调、投喂与 RAG `[理解选型]`

- **Prompt/Tool/Workflow**：先解决行为和业务连接问题。
- **RAG/数据投喂**：知识经常更新、需要引用和权限时优先。
- **微调**：稳定风格、格式、分类或领域行为，且有高质量样本时考虑；不能可靠注入实时事实。
- **PEFT**：LoRA 等参数高效微调，降低训练资源；仍需要数据、评测和部署能力。
- **预训练**：从大规模语料训练基础能力，成本和门槛最高，通常不是应用团队路线。
- **本地模型**：数据主权、离线或规模成本优势明确时选择，同时承担 GPU、推理、升级和安全运维。

Hugging Face 用于模型、数据集、评测和开源生态；检查许可证、模型卡、上下文、语言、工具能力和硬件需求。选择必须在自己的评测集上比较质量、延迟、吞吐、费用和风险。

## 8. 框架与后续路线 `[理解选型]`

LangChain 优点是组件丰富、接入快；代价是抽象变化和调试层级。LangGraph 适合显式状态、持久化和人机协作；简单链不必使用图。DeepAgents 等高级框架先做小型对比实验，不迁移已稳定系统。

关注官方文档、版本说明和可运行示例；社区文章用于理解思路，不作为安全/兼容事实的唯一来源。Prompt 库可参考结构，不直接复制到生产。

建议后续三个月：

1. 第 1 月：用真实业务数据完善客服评测集和 pgvector。
2. 第 2 月：接一个真实只读 API、一个 MCP Server 和持久化 Checkpointer。
3. 第 3 月：上线小流量，建立 Trace、反馈、告警和迭代门禁。

## 常见错误

- 每个案例重新写一套 Agent Loop。
- 文本模型能力边界不清，却宣称支持视觉/语音。
- 第三方回调同步等待长任务。
- 数字人只看演示，不考虑授权、审核和 GPU 队列。
- 微调用来“记住最新企业知识”。
- 没有自有评测集就根据排行榜选模型。
