# 第 10 周轻量指挥台设计

## 目标与边界

为求职演示提供一个 Vue 单页指挥台，把第 9 周 Supervisor 的结构化结果可视化。只包含事件录入、运行结果、Agent 轨迹和审批状态；不增加登录、权限后台、地图 SDK、真实视频或真实调度。

## 方案选择

采用 Vue 3 + Vite 的单页工作台。相比纯静态 HTML，它能练习组件与状态管理；相比完整管理后台，它更容易在一周内理解、测试和演示。前端仅调用 `POST /api/agents/supervisor/invoke`，Vite 开发代理把 `/api` 转发到 FastAPI。

## 组件与数据流

- `App.vue`：页面布局和请求状态。
- `IncidentForm.vue`：收集事件文本、路段、摄像头和资源需求。
- `ResultPanel.vue`：展示风险、预案引用、调度建议、安全结论。
- `AgentTrace.vue`：按执行顺序展示 Agent、尝试次数和成功状态。
- `api.js`：唯一 HTTP 边界，统一抛出可展示错误。

用户提交表单后，前端调用 Supervisor API；加载期间禁用提交按钮；成功后渲染结构化结果；网络或校验失败时显示错误，不伪造结果。

## 视觉原则

采用深蓝应急指挥主题、清晰的状态色和响应式两栏布局。桌面端左侧录入、右侧结果；小屏幕自动变为单列。所有状态同时使用文字和颜色表达，避免只依赖颜色。

## 测试与验收

- Vitest 验证 API 成功、错误处理和关键组件渲染。
- `npm run build` 必须生成生产资源。
- 后端累计测试继续通过。
- 演示场景默认使用 `G65 / QINLING-01`，最终状态必须为“等待人工审批”，且已执行动作为空。
